# Beitragen / Contributing

Danke fuers Interesse, an VoiceAgent mitzuwirken.

## Workflow

1. **Fork** das Repo auf GitHub
2. **Branch** in deinem Fork erstellen (`feature/dein-thema`)
3. **Aenderung** committen mit klaren Commit-Messages (Englisch bevorzugt)
4. **Pull Request** gegen `main` einreichen — beschreibe was und warum

Direkter Push auf `main` ist nicht moeglich (du brauchst einen PR).

## CLA — Beitrags-Vereinbarung

Indem du einen Pull Request einreichst, bestaetigst du:

1. **Urheberschaft**: Du bist Urheber/Eigentuemer des Beitrags ODER hast
   die ausdrueckliche Erlaubnis vom Eigentuemer, ihn beizusteuern.

2. **Du behaeltst die Rechte**: Du bleibst Urheber und Eigentuemer deines
   Beitrags. Wir beanspruchen kein Eigentum.

3. **Nutzungsrecht**: Du gewaehrst dem Maintainer (Atlas) und allen
   nachfolgenden Maintainern dieses Projekts eine **dauerhafte,
   weltweite, kostenlose, nicht-exklusive, uebertragbare und
   unwiderrufliche Lizenz**, deinen Beitrag im Rahmen dieses Projekts
   und seiner Derivate zu nutzen, kopieren, modifizieren, verteilen,
   sublizenzieren und in jeder Form (auch kommerziell) zu verwerten.

4. **Lizenz-Konsistenz**: Dein Beitrag wird unter der gleichen Lizenz
   (siehe `LICENSE`) veroeffentlicht wie das uebrige Repo.

5. **Keine Garantie**: Du leistest deinen Beitrag "as is" — ohne Garantie
   auf Funktionalitaet, Eignung oder Freiheit von Fehlern.

Wenn du diese Punkte nicht akzeptieren kannst, eroeffne bitte ein Issue
zur Diskussion statt einen PR einzureichen.
